# 需要過多・供給不足領域の調査

> 調査日：2026年4月12日

---

## Part 1 — Claude Code講座「需要過多」の原因分析

結論：**需要と供給それぞれに独立した加速要因があり、同時期に発火した**

### 需要側：3つの転換点が重なった

**① 価格の壁が消えた（2025年5〜6月）**

| 時期 | 変化 |
|---|---|
| 2025年5月1日 | MaxプランにClaude Code解放（月$100〜） |
| **2025年6月5日** | **Proプランにも解放（月$20）← 爆発点** |

月$20（約3,000円）で「動くシステムを作れるAI」が使えるようになり、ビジネスパーソン層への普及が一気に起きた。

**② バイブコーディング（Vibe Coding）ムーブメントの着火**

Andrej Karpathy（元Tesla AI責任者）が2025年初頭に提唱した「コードを書かずにAIに委ねる開発スタイル」が日本でも急拡散。**2026年3月には日本AIスキル認定協会が「バイブコーディング検定（初級）」を開始** → 需要が社会制度として固まった。

「プログラミングができなくてもシステムが作れる」という認識が、エンジニア以外の全ビジネスパーソンに「乗り遅れたら負ける」という危機感を生んだ。

**③ 競合ツールからの流入（2025年6月）**

GitHub Copilotが6月から「プレミアムリクエスト数制」に移行 → 使い放題でなくなったユーザーがClaudeへ流入。同タイミングでClaude 4リリース・コーディング能力が大幅向上。「乗り換えるなら今」のタイミングが重なった。

### 供給側：教えられる人が構造的に少ない

| 原因 | 内容 |
|---|---|
| ツール自体が2025年普及 | 経験者の絶対数が少ないまま需要が爆増 |
| 使える人 ≠ 教えられる人 | エンジニアは自分で使うが講師業に転換しない |
| 法人研修の参入障壁 | 専門講師育成3年・LMS構築500万円以上（研修会社試算） |
| 教材陳腐化リスク | アップデート速度が速く、参入を躊躇する講師が多い |

### 需要規模の証拠

Uravation社 Claude Code実践講座（4,000円・3時間・定員300名×2日程）→ **申込者数1,100人突破**
= 定員600名に対し1,100人の申込み。倍率1.8倍以上の需要過多が実証された。

## Part 2 — 類似構造を持つ「需要過多・供給不足」領域マップ

### 選定基準
- 参入障壁が**中程度**（ゼロではないが、3年以上のキャリアは不要）
- 需要の急増が数値で確認できる
- 供給側（教える人・作る人）の絶対数が少ない
- エージェンシー事業と接続できる

---

### 領域①：AIエージェント実装エンジニアリング

**需給データ：**
- エージェントAI関連求人：**2023→2024年で986%増**
- グローバル供給不足率：**50%超**（2026年最大のAIスキルギャップと評される）
- 経産省推計：2040年時点でAI・ロボット利活用人材が**340万人不足**（需要782万人・供給443万人）

**参入障壁の性質：**
- LangChain・n8n等の実装経験が必要
- 「動くシステムを作れる」レベルが求められる
- ただしプログラミング専門家でなくても、ノーコードツールで参入できるルートがある

**エージェンシー事業への接続：**
= このエージェンシーの本丸。実装実績を積むことで単価が急速に上がる領域。

---

### 領域②：n8n / Make による業務自動化構築

**需給データ：**
- 世界のローコード・ノーコード開発市場：2026年に**210億ドル規模**（IDC予測）
- n8nは2025年にシリーズBで$60M調達（評価額$270M）
- 日本国内での「AIエージェントをノーコードで構築できる人」は極めて少ない

**なぜ供給が少ないか：**
- n8nはGitHub Copilotや ChatGPTと違い「自分でサーバを立てる」オプションがある
- セルフホスト版の学習コストが高く、参入を断念する人が多い
- 日本語の実践的な教材がまだ少ない

**参入障壁の性質：**
- セルフホスト版はDockerの知識が必要（中程度の障壁）
- クラウド版（n8n.cloud）なら比較的容易
- 500以上のサービス連携・AIエージェント機能は習得に時間がかかる

**エージェンシー事業への接続：**
ストアカ・ココナラともにn8n構築サービスはほぼ空白地帯。単価30,000〜180,000円レンジで競合なしに近い。

---

### 領域③：プロンプトエンジニアリング（高度・業種特化）

**需給データ：**
- 大企業の約**60%**が2025年中にプロンプトエンジニア採用を計画
- 実務経験保持者：国内推定**3,000〜4,000名**（深刻な人材不足）
- フリーランス平均年収：**1,116万円**（月額単価93万円）
- AI市場は2027年に**1兆1,034億円**規模へ成長予測

**重要な注意点：二極化が進んでいる**
- 「プロンプトを書くだけ」の低スキル → 年収450万円以下に低下
- 業種特化・実装組み合わせのハイスキル → 年収2,000万円のオファーも

**参入障壁の性質：**
- 汎用プロンプト作成は参入障壁が低い（コモディティ化済み）
- **業種知識 × プロンプト最適化** の組み合わせが希少
- EC・製造・医療等の業種専門家が後からAIを学ぶルートが最も差別化できる

---

### 領域④：サイバーセキュリティ

**需給データ（日本）：**
- 不足人員：**約11万人**（調査対象国で最も深刻）
- 需給ギャップ：**97.6%**（供給人材の約2倍の需要）
- 需要増加率：**5倍**（Randstad調査）
- 企業の42%が「必要な人材を確保できない」と回答

**供給が追いつかない理由：**
- 資格取得（CISSP・CEH等）に時間がかかる
- 攻撃手法の進化が速く、知識の陳腐化が早い
- 給与水準が低い企業が多く、人材が大手に集中する
- 2025年から「人員不足」より「スキル不足」が主課題にシフト

**参入障壁の性質：**
- 基礎資格（CompTIA Security+等）は3〜6ヶ月で取得可能
- ただし実務経験なしでは信頼されにくい
- AIを使ったセキュリティ診断・教育というポジションなら比較的早期に参入できる

---

### 領域⑤：AIガバナンス・AI倫理コンサルティング

**需給データ：**
- AIガバナンス関連求人：**1年で150%増**
- AI倫理関連：**125%増**
- EU AI Act施行（2024年）・日本のAI規制議論 → 企業の対応需要が急増

**なぜ供給が少ないか：**
- 「技術 × 法務 × 倫理」の3分野を横断できる人材が希少
- 大学・専門学校でのカリキュラムが整備されていない
- 実務経験を積む場が少ない（新しい職種すぎる）

**参入障壁の性質：**
- 技術的な実装スキルよりも「フレームワークを説明できる」知識型の参入障壁
- EU AI Actの概要 + リスク評価の方法論を習得すれば入門レベルには達せる
- 「AI実装 × ガバナンス整備」をセットで提供できれば高単価

---

### 領域⑥：ローコード・ノーコード開発（PowerApps / Bubble等）

**需給データ：**
- Gartner：2026年にはIT部門外の「市民開発者」がユーザーの**80%以上**を占める
- 世界市場：2026年に**210億ドル**（約3.2兆円）
- 日本のIT人材不足：2030年に最大**79万人**

**参入障壁の性質：**
- PowerApps（Microsoft）はOffice 365環境があれば学習できる
- Bubbleはノーコードながら習得に1〜3ヶ月かかる
- 「作れる人」は増えているが「企業の業務プロセスに合わせて設計できる人」は少ない
- 業種特化（HR・経理・製造等）の知識と組み合わせると差別化できる

**エージェンシー事業への接続：**
Claude Codeやn8nと組み合わせた「フルスタック自動化」として提供できる。

---

### 領域⑦：AI × 動画運用（ショート動画制作・チャンネル運用代行）

**需給データ：**
- YouTube Shorts・TikTok・Instagram Reelsの視聴時間は2022→2023年に大幅増、2026年も継続成長
- AIを使えば30分でショート動画が完成する時代に → 「量産できる人」の需要が急増
- 一方、従来の「手動編集者」は需要減少と危機感

**需給ギャップの構造：**
- 「AIで作れる人」は増えた
- 「AIで量産しながら、チャンネルの戦略・運用まで担当できる人」はまだ少ない
- 企業の動画マーケティング需要に対して、運用代行できる事業者が足りない

**参入障壁の性質：**
- AIツール（Sora・HeyGen・Kling等）の習得に1〜2ヶ月
- チャンネル戦略・アルゴリズム理解には運用実績が必要
- 「撮影ゼロ・テキストから動画」の工程を設計できる人が希少

## Part 4 — エージェンシー戦略への示唆

### 「参入障壁が中程度」が最もおいしい理由

```
参入障壁が低すぎる領域  → すぐにコモディティ化する（汎用ChatGPT相談がその例）
参入障壁が高すぎる領域  → 自社だけでは供給できず、スケールしない
参入障壁が中程度の領域  → 学習コストを払った人だけが参入でき、競合が少ない状態が続く
```

### 複数領域の組み合わせが最強

単一スキルで戦うより、以下のような組み合わせがエージェンシーとして差別化になる：

```
Claude Code × n8n × 業種知識（EC / 製造 / 医療）
     ↓
「AIエージェントを設計し、実装し、教えられる」エージェンシー
     ↓
ストアカで教育 → ランサーズで法人導入支援 → 継続契約（月額顧問）
```

### 今すぐ取るべきポジション

各領域の中で「2026年4月時点で最も供給が少なく、需要が証明されている」のは：

1. **n8n / AIエージェント構築**（ストアカ・ランサーズで競合ほぼゼロ）
2. **Claude Code業種特化実装**（1,100人の需要証明済み）
3. **AIガバナンス × 実装のセット**（規制対応需要が2026年に顕在化）
