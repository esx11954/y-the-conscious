# AIエージェント実装エージェンシー — デモ資産 自動化提案

> `manager-guide-ai-agent-era-2026.md` のフレームワーク・事例を踏まえた、エージェンシー事業向けデモ資産の具体的提案

---

## 前提：デモ資産が達成すべき条件

マニュアルが繰り返す「失敗パターン」から逆算すると、良いデモは：

1. **既存プロセスを自動化するのではなく、再設計を見せる**（Toyota事例の本質）
2. **単一LLMではなく、マルチエージェント連携を可視化する**
3. **Before/Afterの時間差が劇的**（数時間→数分）
4. **見る人が自分の業務に重ねられる**（業界横断的な課題）

---

## デモ提案：優先度順

### ★★★ Priority 1: インバウンドリード処理エージェント

**なぜ刺さるか**: 全業種共通、ROIが即わかる、マルチエージェント構成を自然に見せられる

```
[Webフォーム送信]
    ↓
① 企業調査エージェント
   - 社名でCrunchbase/LinkedIn/HP自動スクレイピング
   - 業種・従業員数・技術スタック・競合を30秒で把握
    ↓
② スコアリングエージェント
   - ICP（理想顧客プロファイル）との適合度を0-100でスコアリング
   - 優先度A/B/Cに自動分類
    ↓
③ パーソナライズ返信エージェント
   - 企業調査結果を踏まえた個別文章を生成
   - スコアAなら「即日返信」、Cなら「自動FAQ返信」
    ↓
④ CRM登録エージェント（HubSpot / Notion等）
    ↓
⑤ 担当者Slack通知（スコアAのみ）
```

**デモの見せ方**: フォーム送信 → 2分後にSlackに「このリード、ICP適合度87点、今すぐ電話推奨」が来る

---

### ★★★ Priority 1: 月次レポート自動生成エージェント

**なぜ刺さるか**: 文書中のB分類の典型例、管理職が即「うちもこれある」と思う

```
[毎月1日 自動起動]
    ↓
① データ収集エージェント
   - Googleアナリティクス・GA4 API
   - 売上データ（Spreadsheet / Shopify等）
   - 顧客サポートチケット数（Intercom等）
    ↓
② 分析エージェント
   - 前月比・前年同月比を計算
   - 異常値・トレンドを検出
   - 「なぜ増えた/減ったか」の仮説を生成
    ↓
③ ドラフト生成エージェント
   - 経営者向け1ページサマリー（マークダウン）
   - 担当者向け詳細版を別途生成
    ↓
④ [Human Review] Slackで担当者に「承認しますか？」を送信
    ↓（承認後）
⑤ メール配信エージェント / Notion自動投稿
```

**ポイント**: ④で止まる構造を見せることで「AIが全部やるわけじゃない、Delegate-Review-Ownだ」を体現できる

---

### ★★☆ Priority 2: 採用スクリーニング自動化

**なぜ刺さるか**: 採用担当者の工数が目に見える形で削減できる

```
[応募メール/フォーム受信]
    ↓
① スクリーニングエージェント
   - 職務経歴書をパース
   - 必須スキル・経験年数をチェックリストと照合
   - 適合度スコアを付与
    ↓
② 一次返信エージェント
   - 不合格：丁寧な不採用通知を自動送信
   - 合格：追加質問フォームを自動送付
    ↓
③ スケジュール調整エージェント（合格者のみ）
   - Cal.com / Calendly APIで面接枠を自動提案
    ↓
④ 担当者に「面接候補者リスト＋スコアカード」をSlack送信
```

---

### ★★☆ Priority 2: 競合・市場監視エージェント

**なぜ刺さるか**: 「エージェントが眠らずに動く」を最もドラマチックに見せられる

```
[毎日深夜3時 自動起動]
    ↓
① 監視エージェント（複数を並列実行）
   - 競合他社のプレスリリースページ
   - 業界メディア（TechCrunch等）の新着記事
   - X（Twitter）の特定キーワードトレンド
   - 競合の求人変化（採用動向からの戦略推測）
    ↓
② 重要度判定エージェント
   - 「自社に影響するか？」を判断
   - High/Medium/Lowに分類
    ↓
③ 要約・示唆生成エージェント
   - 各情報を3行サマリーに圧縮
   - 「だから自社がとるべき行動は？」を付記
    ↓
[朝7時] Slack/メールで担当者に「今日の競合動向ブリーフィング」配信
```

**Toyota事例の再現**: 「担当者が出社した時には、すでに対処が終わっている」を体験させる

---

### ★☆☆ Priority 3: SNS/コンテンツ運用エージェント

**なぜ刺さるか**: 小規模チームでも見える効果、ただし「完全自動投稿」はリスクがあるため要注意

```
① トレンド収集エージェント（毎日）
② コンテンツ提案エージェント（5案を一覧化）
③ [Human Review] 担当者が1案を選択
④ 画像プロンプト生成 → DALL-E/Midjourney API
⑤ [Human Approval] 最終承認
⑥ 投稿スケジューリングエージェント
```

## デモ設計の共通原則

| 要素 | 実装方針 |
|------|---------|
| **可視化** | n8n / Langfuse等でエージェント間の流れをリアルタイムに表示 |
| **Human-in-the-loop** | 必ずSlack承認ステップを入れ「AIが暴走しない」を見せる |
| **Before/After数値** | 「従来X時間→今Ymin」を画面に出す |
| **エージェント名の表示** | 「検知エージェント → 分析エージェント → 報告エージェント」と明示的に命名して見せる（Toyota事例の構成を意識） |

---

## 技術スタックの実例候補

| レイヤー | 選択肢 |
|---------|-------|
| オーケストレーション | n8n（ノーコード可視化が強い）/ LangGraph |
| LLM | Claude Sonnet 4.6（ツール使用・長文処理が強い） |
| Human-in-the-loop通知 | Slack Webhooks |
| CRM連携 | HubSpot API / Notion API |
| 監視ダッシュボード | Langfuse / Langsmith |

---

## 最優先デモ推奨

**インバウンドリード処理エージェント** か **月次レポート自動生成エージェント** を最初に作るべき。

- 前者はBtoB営業文脈で誰でも理解でき、即座のROIを示せる
- 後者はDelegate-Review-Ownモデルを体験形式で見せられるため、`manager-guide-ai-agent-era-2026.md` とセットで訴求できる
