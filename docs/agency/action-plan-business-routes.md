# AIエージェント実装エージェンシー — 事業成功ルート & アクションプラン

> 策定日：2026年4月12日
> 前提資料：market-research-platform-ai-consulting.md / market-research-demand-supply-gaps.md / demo-assets-ec-operator.md

---

## 基本方針：3ルートは「選択肢」ではなく「時系列」

| ルート | 内容 | 時期 |
|---|---|---|
| Route A | 教育（ストアカ講座）で顧客の悩みを収集・実績構築 | Month 1〜3 |
| Route B | 業種特化（EC）デモで法人契約を獲得・単価引き上げ | Month 3〜6 |
| Route C | 実装事例をコンテンツ化してインバウンドを作る | Month 6〜（副産物として並走） |

---

## フライホイール構造

```
          ┌─────────────────────────────┐
          ↓                             ↑
   ストアカ講座で               実装事例がコンテンツになる
   顧客の悩みを収集                     ↑
          ↓                             │
   悩みを業種別に分類          法人契約の成功事例を
          ↓                   note/Xで発信
   業種特化デモを構築                   ↑
          ↓                             │
   デモで法人を獲得  ─────────────────→┘
          ↓
   実装 → 顧問 → 継続収益
```

一度回り始めると、各要素が互いを強化する。

## フェーズ別アクションプラン

---

### Phase 0（今すぐ〜Month 1）：アセット整備

**やること**

- [ ] ストアカ講座ページを作成（「Claude Code × 業務自動化 入門90分」、1,000〜2,000円）
- [ ] ランサーズのプロフィールを整備
- [ ] ランサーズ「Consul Partners for AI」に登録申請
- [ ] Xアカウントで発信を週3本ペースで開始（調査結果がそのまま素材になる）

**コスト：** ほぼゼロ（時間のみ）

**得られるもの：** 集客の導線が3本立つ（ストアカ・ランサーズ・X）

---

### Phase 1（Month 1〜3）：検証と実績構築

**ストアカ：顧客の悩みを集める**

```
1,000〜2,000円の入門講座を開催
    ↓
受講者20〜30人を集める
    ↓
「どの業務で困っているか」を直接ヒアリング
    ↓
最も多い悩みパターンを特定する
```

**ランサーズ・ランサーズ：n8n構築サービスを出品**

- タイトル例：「業務自動化エージェント構築（n8n使用）」
- 競合がほぼいないため、レビューゼロでも選ばれやすい
- 最初の3件は赤字覚悟で実績を積む

**この期間で得られるもの**

- 顧客の「生の悩み」データ
- ストアカ・ランサーズのレビュー（信頼のシグナル）
- 実際に動いた自動化の事例（= 次フェーズのデモ資産）

**収益目標：** 月5〜15万円（講座 + 初期構築案件）

---

### Phase 2（Month 3〜6）：業種特化と単価引き上げ

Phase 1のヒアリングで最も多かった業種に全集中。**EC事業者向けデモ資産はすでに設計済み**（demo-assets-ec-operator.md）。

**EC事業者に集中する場合のフロー**

```
ストアカで「EC事業者向けAIエージェント講座（3時間）」を5,000〜8,000円で出品
         ↓
受講者の中から「作ってほしい」という人を個別対応
         ↓
カゴ落ち回収・競合価格監視デモを実際に動かして見せる
         ↓
「まず1ヶ月試しに作りましょう」で30〜50万円の初期構築
         ↓
「運用・改善を続けるなら月額5〜10万円で」と顧問化
```

**デモで見せる最優先2本（すでに設計済み）**

1. カゴ落ち回収エージェント — フォーム放棄→2分後にパーソナライズメールが届く体験
2. 競合価格監視エージェント — 「昨夜、競合Aがこの商品を¥800値下げしました」をSlackで受け取る体験

**収益目標：** 月20〜50万円（構築1〜2件 + 顧問2〜3社）

---

### Phase 3（Month 6〜12）：レバレッジをかける

3つの追加要素を投入する。

**要素①：IT導入補助金の活用**

中小企業はAIエージェント導入費用を補助金で賄える。

| 補助金 | 概要 | 補助率 |
|---|---|---|
| IT導入補助金（経産省） | ソフトウェア導入・業務効率化 | 最大75% |
| 小規模事業者持続化補助金 | 販路開拓・業務効率化 | 最大2/3 |

「補助金を使えば実質負担10万円でAIエージェントを導入できます」というトークが法人営業で強力な武器になる。申請サポートまでできると引き合いがさらに増える。

**要素②：ケーススタディのコンテンツ化**

```
EC事業者Aのカゴ落ち回収率が3%→11%に改善
         ↓
Before/After数値をまとめたnote記事・X投稿
         ↓
「うちにも同じことをやってほしい」というインバウンドが来る
         ↓
営業コストゼロで次の案件が決まる
```

**要素③：ツールパートナー制度の活用**

n8nはパートナープログラムを持っている。認定パートナーになると：
- 紹介案件が来る導線ができる
- クライアントへの信頼性シグナルになる
- 割引価格でライセンス提供できる

**収益目標：** 月60〜120万円（顧問4〜6社 + 講座売上 + 新規構築）

---

### Phase 4（Month 12〜）：プロダクト化とスケール

「時間を売る」から「仕組みを売る」へ移行する。

**プロダクト化の例**

```
「EC事業者向けカゴ落ち回収エージェント」パッケージ
    ├── 初期セットアップ費：15万円
    ├── n8nテンプレート提供（設定済み）
    ├── 90分のオンボーディング講座
    └── 月額保守：3万円

→ テンプレートを作れば1件あたりの工数が激減
→ 月5件販売で初期収益75万円 + 保守15万円
```

**パートナーエージェンシー化**

```
ストアカ受講者の中から実装できる人材を育成
         ↓
案件の一部を任せる（自分はディレクションに専念）
         ↓
1人でできる案件数の3〜5倍をこなせる
```

## 差別化の壁：Level別に積み上げる

単に「AIが使える」では差別化にならない。以下を順番に積む：

```
Level 1：使える              （誰でもできる）
Level 2：教えられる          （ストアカ実績）
Level 3：業種知識がある      （EC特化）       ← ここで競合が激減する
Level 4：実装事例がある      （数字で話せる）
Level 5：補助金まで支援できる                 ← ここで参入障壁が完成する
```

Level 3以上に到達した時点で、価格競争から抜け出せる。

---

## 戦略的リスクと対処

| リスク | 内容 | 対処 |
|---|---|---|
| Claude/n8nのアップデートで知識が陳腐化 | ツールが急変した時に価値が消える | 「特定ツール」ではなく「業種課題の解決」に軸足を置く |
| 受注が集中して品質が落ちる | 1人でさばける限界を超える | Phase 3でパートナー育成を早めに始める |
| 大手エージェンシーが参入 | 資金力で押される | 業種特化の深さで対抗（大手は横広がりを好む）|
| 補助金制度が変わる | 売り文句が使えなくなる | あくまで「ROIで元が取れる」を主軸にする |

## 意思決定ツリー：今どこから始めるか

```
Q1: 今すぐ収益が必要か？
    Yes → Route B（EC特化デモ）を先行、講座は後回し
    No  → Route A（ストアカ講座）から市場検証

Q2: EC以外に詳しい業種があるか？
    Yes → その業種向けデモを先に作る（EC調査結果は流用可）
    No  → ECで始める（デモ資産がすでにある）

Q3: 発信（X・note）を続けられるか？
    Yes → Phase 1からRoute Cを並走させる
    No  → Phase 2のケーススタディができてから始める
```
