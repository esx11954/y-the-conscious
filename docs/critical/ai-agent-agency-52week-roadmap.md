## Phase 0 — 武器を磨く

**期間**: 1〜4週目 | **投下時間**: 週20h → 合計80h

:::info Goal
自分たちで「1案件を完遂できる状態」を作る
:::

### 1〜2週目（40h）— 技術キャッチアップ＋自社実験

- **あなた**: Claude / ChatGPT / Dify / n8n を使い倒す。特にMCPの概念と「何ができるか」を体感する。非エンジニアでもDifyやn8nならノーコードでAI×Slack/Gmail連携が組める
- **エンジニア**: MCPサーバーの構築を実際にやる。Claude Code + MCP SDK で、Slack→タスク自動作成、Gmail→CRM連動など3つのプロトタイプを作る
- **2人で**: お互いの本業の業務フローを棚卸しして、AIエージェントで自動化できるポイントを10個リストアップ→上位3つを実際に自動化してみる

### 3〜4週目（40h）— デモ資産を作る＋ポジショニング決定

- 実際に自動化した3つを「ビフォーアフター動画」にする（Loom等で3分以内）。これが最強の営業ツールになる
- 屋号とLP（ランディングページ）を用意。バイブコーディングで半日で作れる。「AIで業務を自動化」ではなくもっと具体的に。例：『問い合わせ対応を80%自動化するAIエージェント構築』
- ターゲット業種を1つに絞る（後述の選定基準を参照）。最初は広げない。1業種で実績を作ることが最優先

## Phase 1 — 最初の3件を獲る

**期間**: 5〜12週目 | **投下時間**: 週20h → 合計160h

:::info Goal
実績0→3件。単価30〜50万円のプロジェクトを完遂する
:::

### 5〜8週目 — 初期営業（泥臭くやる）

- 本業の人脈・取引先に「AIで業務自動化の実験をしてる。無料or格安でやらせてほしい」と声をかける。最初の1件は¥0〜10万円でいい。実績とケーススタディを買う
- SNS（X / LinkedIn）でデモ動画を発信。『中小企業の◯◯業務をAIで自動化してみた』系のコンテンツは現在バズりやすい。週2〜3投稿を習慣化
- 地元の商工会議所・異業種交流会に顔を出す。「AIで何ができるか」を10分で話せるミニセミナーを無料で提供。ここが一番の受注チャネルになりうる
- ココナラ・ランサーズ等で「ChatGPT/AI活用コンサルティング」を出品。初期は1時間5,000円の相談から始めてもいい。ここでの目的は「顧客の生の悩み」を大量に集めること

### 9〜12週目 — 初案件を納品＋型化

- 最初の案件を全力で納品。工数は赤字でもいい。ただし必ず①クライアントの声（動画 or テキスト）②数値的成果（◯時間削減、◯%コスト削減）を取得する
- 納品プロセスを「型」にする：ヒアリングシート→業務フロー図→エージェント設計書→構築→テスト→引き渡し＋運用マニュアル。この型があれば2件目以降の工数が半分になる
- 1件目の成果をケーススタディ記事にしてLP・SNSに掲載

## Phase 2 — 単価を上げて月商100万を作る

**期間**: 13〜24週目（約3〜6ヶ月目）| **投下時間**: 週20h → 合計240h

:::info Goal
月商50〜100万円を安定させる。単価50〜150万円
:::

### 13〜18週目 — サービスを3段階パッケージ化

- 松竹梅で価格設定。①スターター：単一業務のAI自動化（30〜50万円）②スタンダード：部門横断3〜5エージェント連携（80〜150万円）③エンタープライズ：全社AI戦略設計＋実装（200万円〜）
- 月額保守プラン（3〜8万円/月）を必ずセットで提案。エージェントのチューニング、プロンプト改善、新機能追加。これがストック収益になる
- ケーススタディが3件溜まったら、それを武器にターゲット業種の企業にDM営業。具体的数値（◯時間削減、◯万円コスト削減）があれば返信率は格段に上がる

### 19〜24週目 — リード獲得の仕組み化

- 無料ウェビナー月1回開催（ターゲット業種向け）。Zoomで30分。『◯◯業界のAI活用最前線〜問い合わせ対応を80%自動化した実例〜』。これが最も費用対効果の高い集客チャネル
- ウェビナー参加者に無料診断（30分）を提供→課題が具体化→提案→受注。このファネルを毎月回す
- 紹介プログラムを設計。既存クライアントからの紹介で成約したら◯◯をバックするなど。B2Bの最強チャネルは紹介

## Phase 3 — ストック型に転換する

**期間**: 25〜52週目（6ヶ月〜1年目）

:::info Goal
受託依存を脱し、月額課金の収益基盤を作る
:::

### 25〜36週目 — 受託で得た知見をプロダクト化

- 複数案件で共通するパターンをSaaS/テンプレート化。例：不動産業界なら「物件問い合わせ自動応答＋内見予約＋追客メール自動化」のパッケージをクラウドサービスとして月額課金で提供
- 受託で稼ぎながら、エンジニアパートナーがプロダクト開発に徐々にシフト。あなたは営業・CS・コンテンツに集中する分業体制を固める
- この段階で本業からの独立を判断。月商が本業年収の50%を安定的に超えたら、独立の合理性が出る

### 37〜52週目 — スケールの種を蒔く

- 業種特化パッケージの月額顧客が10社を超えたら、次の業種にも横展開。「型」は同じで業種知識だけ入れ替える
- エージェントコマース対応（UCP/MCP）をサービスラインに追加。EC事業者向けに『AIショッピングエージェントに見つけてもらえる商品データ整備サービス』を提供
- AIエージェント人材育成コンテンツ（動画講座・eラーニング）を副次的に販売。受託案件の裏側ノウハウを教材にする。これは限界費用ゼロの収益源

## Target Industries（推奨ターゲット業種）

| 業種 | スコア | 推奨理由 |
|-----|-------|---------|
| 不動産 | 92/100 | 問い合わせ対応・物件紹介・内見予約・追客が定型化しやすい。IT苦手な中小不動産会社が多く、単価も取りやすい。実例：米国では1社あたり$13,000のリード自動振り分けシステムで月200件以上を5分のレビューで処理 |
| 士業（税理士・社労士・弁護士事務所） | 88/100 | 顧問先からの定型質問対応、書類作成補助、期限管理。デジタル化が遅れている業界でAI導入への潜在需要が高い。規制知識が参入障壁になり、一度入ると替えられにくい |
| ECショップ運営 | 85/100 | カスタマーサポート自動化、商品説明文生成、在庫連携、レビュー分析。UCP/MCPプロトコル対応需要が今後急増する。エージェントコマースへの対応がそのまま将来のPhase 3に繋がる |
| クリニック・歯科医院 | 82/100 | 予約管理、問い合わせ対応、リマインド配信。開業医は経営判断が早く、IT投資への抵抗が比較的少ない。1院あたりの単価は低めだが、横展開しやすい |

## Pricing Model（料金体系）

### スターター — ¥300,000〜500,000

| 項目 | 内容 |
|-----|------|
| 月額保守 | 3万円/月 |
| スコープ | 単一業務の自動化（例：問い合わせ自動応答） |
| 納期 | 2〜3週間 |
| あなたの工数 | ヒアリング・設計・提案（10h） |
| エンジニア工数 | 構築・テスト・デプロイ（20h） |

### スタンダード — ¥800,000〜1,500,000

| 項目 | 内容 |
|-----|------|
| 月額保守 | 5万円/月 |
| スコープ | 部門横断3〜5エージェント（例：営業部門の一気通貫自動化） |
| 納期 | 4〜6週間 |
| あなたの工数 | ヒアリング・設計・PM・納品（25h） |
| エンジニア工数 | 構築・連携・テスト（50h） |

### エンタープライズ — ¥2,000,000〜

| 項目 | 内容 |
|-----|------|
| 月額保守 | 8万円〜/月 |
| スコープ | 全社AI戦略設計＋複数部門への実装 |
| 納期 | 2〜3ヶ月 |
| あなたの工数 | 戦略立案・PM・報告（50h） |
| エンジニア工数 | 設計・構築・運用基盤（100h+） |
